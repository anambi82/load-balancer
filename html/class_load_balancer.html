Detailed Description

Orchestrates the simulation of a dynamically scaling server pool.

On each clock cycle the LoadBalancer:

  1. Probabilistically generates a new incoming request.
  2. Advances all busy servers by one clock cycle, completing requests where due.
  3. Distributes queued requests to idle servers (round-robin over available slots).
  4. Evaluates scaling thresholds and adds or removes servers as needed.

Requests originating from blocked IP ranges are silently dropped and logged. Autoscaling is gated by a configurable cooldown period to prevent thrashing.

void LoadBalancer::run ( )

Runs the main simulation loop for the configured number of clock cycles.

Each cycle: generates requests, processes servers, distributes work, and checks scaling. Logs a status snapshot every totalRunTime/20 cycles.
